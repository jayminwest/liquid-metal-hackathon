# Data Directory

This directory contains local knowledge storage for **basic-memory**.

## Structure

```
data/
├── entities/           # Knowledge entities (Markdown files)
├── projects/           # Project-based organization
└── .basic-memory/      # SQLite index (auto-generated by basic-memory)
```

## Entity Format

Each entity is stored as a Markdown file with:

1. **Frontmatter** - Metadata
2. **Content** - Main description
3. **Observations** - Facts organized by category
4. **Relations** - Links to other entities via WikiLinks

### Example Entity

```markdown
---
title: LiquidMetal Framework
tags: [platform, ai, paas]
source: documentation
created: 2025-11-08T12:00:00Z
url: https://docs.liquidmetal.ai
---

# LiquidMetal Framework

Platform-as-a-Service for building AI-native applications with built-in infrastructure components.

## Observations

- [feature] Actor-based compute isolation with 10GB storage per user
- [feature] SmartBucket provides RAG-enabled document storage
- [feature] SmartSQL offers natural language database queries
- [use_case] Ideal for hackathon rapid prototyping #hackathon
- [fact] Built on Raindrop framework

## Relations

- built_on [[Raindrop]]
- provides [[SmartBucket]]
- provides [[SmartSQL]]
- provides [[SmartMemory]]
- related_to [[AI Platforms]]
```

## Usage with basic-memory

### Initialize

```bash
npx @basicmachines/basic-memory --dir ./data
```

### Create Entity

```bash
# Via MCP tool
write_note({
  path: "entities/topic-name.md",
  content: "..."
})
```

### Query

```bash
# Via MCP tool
query({ query: "AI platforms" })
```

### Build Context

```bash
# Via MCP tool
build_context({
  paths: ["entities/liquidmetal-framework.md"],
  depth: 2
})
```

## Sync with Raindrop

All entities are automatically synced to Raindrop Working Memory via `api/knowledge/sync.ts`:

- **Write**: Creates entity locally + stores in Raindrop
- **Read**: Combines local graph + Raindrop semantic search
- **Update**: Syncs changes bidirectionally

## File Naming Convention

Entity filenames use lowercase with hyphens:

- "LiquidMetal Framework" → `liquidmetal-framework.md`
- "AI Platforms" → `ai-platforms.md`
- "Knowledge Graphs" → `knowledge-graphs.md`

## Version Control

This directory is git-friendly:

- Markdown files are human-readable diffs
- Can track knowledge evolution over time
- Collaborative editing possible
- `.basic-memory/` directory should be gitignored (binary SQLite DB)

## Best Practices

1. **One Topic Per File** - Keep entities focused
2. **Rich Relations** - Link entities liberally with `[[WikiLinks]]`
3. **Categorized Observations** - Use consistent categories
4. **Tag Generously** - Tags enable discovery
5. **Source Attribution** - Always note where knowledge came from
